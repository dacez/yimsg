## 1. 总体约束规则

### 1.1 表结构已固化

数据库表结构已固化，任何 schema 变更（新增/删除/修改表或列）必须先确认，未经批准不得执行。

### 1.2 SQL 约束

| 规则 | 说明 |
|------|------|
| 禁用 JOIN | 所有跨表查询在应用层完成 |
| 最小化事务 | 优先使用单条 SQL 语句，仅在必须保证原子性时使用事务 |
| 禁用自增主键 | 所有 ID 使用 Snowflake 生成，保证全局唯一且可排序 |
| 时间戳统一毫秒 | 所有 `created_at`/`updated_at`/`send_time` 等字段均为毫秒级 Unix 时间戳 |
| 空串代替 NULL | 所有 TEXT 字段使用 `NOT NULL DEFAULT ''`，避免 NULL 语义歧义 |
| 0 代替 NULL | 所有互斥 ID 字段（`friend_uid`/`group_id`/`org_id`/`to_uid`/`child_tag_id`/`uid`）使用 0 表示"无" |

### 1.3 存储引擎约束

| 规则 | 说明 |
|------|------|
| SQLite WAL 模式 | 单写多读（1 writer + N readers per shard） |
| 纯 Go 驱动 | 使用 `modernc.org/sqlite`，无 CGO 依赖 |
| 分片架构 | 所有数据按路由键分片，`hash(key) % N → shard` |

### 1.4 数据一致性约束

| 规则 | 说明 |
|------|------|
| 无跨分片事务 | 跨分片操作允许部分失败，由业务层容忍或重试 |
| 幂等写入 | 关键写入使用 `INSERT OR IGNORE` 或 `ON CONFLICT DO NOTHING` 保证幂等 |
| 单调递增序列 | `messages.seq`、`conversations.seq` 共用 `messages_version.max_seq`；`contacts.seq`、`blocklist.seq`、`mutelist.seq` 在各自 uid 分片内单调递增；`org_tag.seq`、`org_tag_item.seq` 共用 `org_version.max_seq` 在 org 分片内单调递增 |

### 1.5 显示数据约束

| 规则 | 说明 |
|------|------|
| 客户端 Cache 管理 | 显示数据（昵称/头像）由客户端 uidCache/groupCache 管理 |
| 服务端维护排序/搜索投影 | 客户端调用 `get_user_infos` / `get_group_infos` 拉取展示资料时，服务端会在 `internal/service/user.go` / `internal/service/group.go` 中刷新调用方联系人的 `sort_key`（无备注时）与 `search_text`（含昵称/群名，始终刷新），用于保持联系人分页排序与搜索稳定；会话预览仍由消息 / 会话写链路维护 |

## 2. 分片架构

路由规则：`hash(key) % N → shard`

| 分片组 | 路由键 | 包含的表 | 用途 |
|--------|--------|----------|------|
| uid | uid (int64) | user_info, contacts, contacts_version, blocklist, blocklist_version, messages, messages_version, conversations, mutelist, mutelist_version, user_session | 用户个人数据 |
| username | username (string) | user_lookup | 用户名→UID 映射 |
| token | token (string) | session | 认证令牌 |
| group | group_id (int64) | group_info, group_member | 群属性和成员 |
| org | org_id (int64) | org_tag, org_tag_item, org_version | 组织 tag 图（组织即根 tag）与同步版本 |

## 4. 垃圾回收总览

所有 GC 循环在后台 goroutine 中运行，使用游标分批扫描，跨分片并发。

| GC 类型 | 目标表 | 策略 | 默认间隔 | 批大小 |
|---------|--------|------|----------|--------|
| Session GC | session, user_session | 删除过期 session + 清理孤立 user_session | 3600s | 500 |
| Message GC | messages | 每用户默认保留最近 100000 个消息 seq 保留范围；不推进 `messages_version.gc_safe_seq`，历史缺口允许漏同步 | 3600s | 500 |
| Conversation GC | conversations | 每用户默认保留最近 10000 个会话 `seq` 窗口，窗口外会话行可直接删除；共用 `messages_version`，不推进 GC 水位线 | 3600s | 500 |
| Contact GC | contacts, contacts_version | 物理删除软删除记录，升 gc_safe_seq 水位线 | 3600s | 500 |
| Blocklist GC | blocklist, blocklist_version | 物理删除已解除屏蔽 tombstone，升 gc_safe_seq 水位线 | 3600s | 500 |
| Mute GC | mutelist, mutelist_version | 物理删除 `status=0xff` 的关闭记录，升 gc_safe_seq 水位线 | 3600s | 500 |
| Org GC | org_tag, org_tag_item, org_version | 物理删除 tag 图 tombstone（节点与边），升 gc_safe_seq 水位线（org 分片组） | 3600s | 500 |
| User GC | user_lookup | 清理孤立用户名映射（无对应 user_info 记录） | 3600s | 500 |
