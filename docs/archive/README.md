## 不应进入归档的内容

以下内容不进入 `archive/`，避免归档目录变成新的垃圾桶：

1. **当前实现说明**：应合并到对应权威设计文档。
2. **仍需执行的 TODO / 路线图**：应进入当前评估或 issue，不放入历史归档。
3. **旧接口、旧 schema、旧 UI 流程**：研发阶段不维护旧格式兼容，失效内容直接删除。
4. **与权威文档重复的完整表格**：只保留必要历史结论，当前矩阵以权威文档为准。
5. **临时调试记录、会议纪要或个人笔记**：若没有长期追溯价值，不提交到仓库文档。
