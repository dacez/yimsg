## 2. 本轮治理结果

| 原问题 | 当前处理结果 |
|---|---|
| `接口总览.md` 缺少内联导航 | 已在文档头部增加快速导航，覆盖 SDK / 服务端映射、对外接口、推送矩阵、附录和落地清单 |
| 文档头部模板不统一 | `docs/**/*.md` 统一校验“主要对照 / 最后复核 / 触发更新 / 入口关系”四行格式 |
| `测试方案.md` 硬编码测试数量 | 已删除具体数量，测试统计统一由 `tools/check_docs_consistency.sh` 动态输出 |
| 跨文档链接缺乏自动校验 | 一致性脚本已增加 Markdown 相对链接断链检测 |
| 插件约束分散 | `AGENTS.md` 只保留执行流程和工具约定，插件设计约束指向 `docs/插件架构方案.md` |
| SDK 与服务端接口文档交叉重复 | `接口总览.md` 保留服务端 action、字段和硬约束；`sdk接口说明.md` 保留客户端调用、前置条件和返回类型，并在 `client_config` 交叉点单向链接 |
| 版本标注方式不一致 | 当前文档统一使用“研发阶段，以当前分支代码为准”表达当前状态，不引入不存在的版本号概念 |
| `最后复核` 日期易遗漏 | 一致性脚本会对变动但未更新复核日期的文档输出 warning，不阻断本地迭代 |
| `archive/` 边界不清 | `docs/archive/README.md` 已补充“不应进入归档”的反向规则 |
| 同步机制说明分散 | 新增 `同步机制方案.md` 作为消息、通讯录、屏蔽列表、免打扰和会话同步的共同入口，接口、SDK、领域专题只保留链接和领域差异 |

## 4. 后续维护关注点

1. 新增文档时必须补齐四行头部模板，并从对应索引建立入口关系。
2. 修改 action、schema、DAL、SDK API 或测试结构时，优先同步权威文档，再运行 `./tools/check_docs_consistency.sh`。
3. 只改文档、索引或归档规则时，可只运行文档一致性校验；涉及实现、测试或构建脚本时运行 `./tools/run_all_tests.sh`。
4. 历史材料只在确有追溯价值时进入 `docs/archive/`；当前实现、待办事项、旧接口或重复矩阵不进入归档。
5. 每次更新评分时，须同步说明分数变化原因，并将已完成治理记录移入 [`archive/文档治理历史.md`](archive/文档治理历史.md)（如需保留历史）。
