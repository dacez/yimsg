## 帧格式

核心业务 WebSocket 帧使用 protobuf 定义 body。帧格式为 `magic:uint8('M') + codec:uint8(bitfield) + reserved:uint8(0) + checksum:uint8(CRC-8) + size:uint16 + request_id:uint64 + type:uint16 + body`，header 为 16 字节，`size` 最大是 `65519`；`codec` bit0 表示大小端，bit1-4 表示 version，bit5-7 保留并与后续 `reserved` 字节连续；`type=0` 是无效值；action enum 命名统一为 `TYPE_ACTION_*`，通知 enum 命名统一为 `TYPE_NOTIFY_*`；`request_id=0` 且 `type` 位于通知段表示服务端通知。

## 生成边界

- Go：action 入方向生成 `ActionService` 接口 + `DispatchActionFrame(svc, info, frame)`；notification 出方向生成 `NewXxxNotificationFrame` / `EncodeNotificationFrame`。
- TS：action 出方向生成无状态函数（如 `login(transport, req)`）；notification 入方向生成 `NotificationHandler` 接口 + `dispatchNotificationFrame(handler, frame)`。
- 业务逻辑（Go service）、SDK 公开接口、DataGateway、缓存、状态机、`msg_id` 生成仍然手写；fanout（异步任务队列）/ notify / DB 写入属于 service，不属于 dispatch。

## Action 字段明细

### `register`

- 请求：`RegisterRequest`
  - `username`（`string`，必填）：用户名
  - `password`（`string`，必填）：密码
  - `nickname`（`string`，可选）：昵称
- 响应：`RegisterResponse`
  - `uid`（`number`，必填）：注册后的用户 ID

### `login`

- 请求：`LoginRequest`
  - `username`（`string`，必填）：用户名
  - `password`（`string`，必填）：密码
- 响应：`LoginResponse`
  - `uid`（`number`，必填）：登录用户 ID
  - `token`（`string`，必填）：新会话 token
  - `client_config`（`ClientConfig`，必填）：客户端运行配置

### `authenticate`

- 请求：`AuthenticateRequest`
  - `token`（`string`，必填）：会话 token
- 响应：`AuthenticateResponse`
  - `uid`（`number`，必填）：鉴权用户 ID
  - `client_config`（`ClientConfig`，必填）：客户端运行配置

### `logout`

- 请求：`LogoutRequest`
  - `token`（`string`，可选）：会话 token
- 响应：`LogoutResponse`
  - 无业务字段。

### `ping`

- 请求：`PingRequest`
  - 无业务字段。
- 响应：`PingResponse`
  - 无业务字段。

### `update_user_info`

- 请求：`UpdateUserInfoRequest`
  - `nickname`（`string`，可选）：昵称
  - `avatar`（`string`，可选）：头像 URL

### `update_password`

- 请求：`UpdatePasswordRequest`

- 响应：`UpdatePasswordResponse`
  - 无业务字段。

### `get_user_infos`

- 请求：`GetUserInfosRequest`

- 响应：`GetUserInfosResponse`

### `search_user`

- 请求：`SearchUserRequest`
  - `username`（`string`，必填）：用户名
- 响应：`SearchUserResponse`
  - `profile`（`UserInfo`，可选）：用户资料

### `add_friend`

- 请求：`AddFriendRequest`
  - `friend_uid`（`number`，必填）：好友用户 ID
  - `remark_name`（`string`，可选）：备注名
- 响应：`AddFriendResponse`
  - `seq`（`number`，必填）：通讯录同步序号

### `accept_friend`

- 请求：`AcceptFriendRequest`
  - `friend_uid`（`number`，必填）：好友用户 ID；调用者必须是该请求的接收方（自身记录为 CONTACT_STATUS_PENDING_INCOMING），否则返回 ERROR_CONFLICT
- 响应：`AcceptFriendResponse`
  - 无业务字段。

### `reject_friend`

- 请求：`RejectFriendRequest`
  - `friend_uid`（`number`，必填）：好友用户 ID；调用者必须是该请求的接收方（自身记录为 CONTACT_STATUS_PENDING_INCOMING），否则返回 ERROR_CONFLICT
- 响应：`RejectFriendResponse`
  - 无业务字段。

### `delete_friend`

- 请求：`DeleteFriendRequest`
  - `friend_uid`（`number`，必填）：好友用户 ID
- 响应：`DeleteFriendResponse`

### `update_remark`

- 请求：`UpdateRemarkRequest`
  - `target`（`ContactTarget`，必填）：联系人目标；通讯录语义使用 ContactTarget，避免与会话目标混用
  - `remark_name`（`string`，必填）：备注名
- 响应：`UpdateRemarkResponse`
  - 无业务字段。

### `get_contacts`

- 请求：`GetContactsRequest`
  - `status`（`number`，可选）：联系人状态过滤；不填表示不过滤，显式传 0 返回 ERROR_INVALID_ARGUMENT
  - `targets`（`ContactTarget[]`，可选）：联系人目标过滤；单个查询传 1 个元素，命中时忽略 page 分页
  - `page`（`PageQuery`，可选）：展示分页游标；friend 状态按 sort_key/拼音排序，pending 状态按 seq 倒序，游标对客户端不透明
- 响应：`GetContactsResponse`
  - `contacts`（`Contact[]`，可选）：通讯录条目
  - `page`（`PageInfo`，必填）：展示分页信息

### `get_contact_count`

- 请求：`GetContactCountRequest`
  - `status`（`number`，必填）：统计状态；必须显式传入合法非 0 状态，例如 CONTACT_STATUS_PENDING_INCOMING 表示待我处理的好友申请数，CONTACT_STATUS_FRIEND 表示好友/收藏群数量
- 响应：`GetContactCountResponse`
  - `total`（`number`，必填）：指定状态的通讯录条目总数

### `sync_contacts`

- 请求：`SyncContactsRequest`
  - `last_seq`（`number`，可选）：客户端最后同步序号；增量同步使用 seq 游标，与通讯录展示分页 offset/limit 分离
  - `limit`（`number`，可选）：同步条数
  - `rebuild`（`boolean`，可选）：是否强制重建本地视图
- 响应：`SyncContactsResponse`
  - `contacts`（`Contact[]`，可选）：增量通讯录条目
  - `has_more`（`boolean`，必填）：是否还有更多增量条目（达到 limit 时为 true）
  - `cursor_seq`（`number`，必填）：下一次增量同步应使用的 last_seq 游标，等于本批最大 seq；本批为空时为 0
