## 1. 定位

Yimsg 服务端是单个 Go 二进制 + 本地 SQLite 分片文件，不依赖外部数据库/缓存服务；前端构建产物是静态文件，由服务端直接挂载。部署形态是**单机 systemd 常驻**，不使用容器、不使用负载均衡多实例（在线连接注册表和消息 fanout 是单进程内存态，详见 `internal/online/online.go`，因此天然是单机架构）。

本文只覆盖“把代码部署到一台已购买的 Linux 云服务器”这件事本身，域名解析、云厂商安全组/防火墙规则由使用者在对应控制台自行操作，本文档只记录需要放行的端口。

## 2. 当前实现

| 项 | 值 |
|---|---|
| 服务器操作系统 | Debian 13 (trixie) x86_64 |
| 服务器 IP | `43.131.244.40`（云厂商分配，重建实例会变化，以实际控制台显示为准） |
| SSH 别名（本机 `~/.ssh/config`） | `yimsg-server` |
| 部署路径 | `/opt/yimsg/`（`server` 二进制、`config.toml`、`web/` 聊天 App 产物、`website/` 官网静态资源、`data/` 数据目录） |
| 运行账号 | 系统账号 `yimsg`（`useradd --system --shell /usr/sbin/nologin`），非 root |
| 监听端口 | `0.0.0.0:443`（生产直连 HTTPS，证书由服务端进程读取） |
| TLS 证书 | `/etc/ssl/certs/yimsg.pem`、`/etc/ssl/certs/yimsg.key`；本地开发 / 自动化测试不配置这两项，继续使用 HTTP |
| URL 结构 | HTTPS 根路径 `/` 是官网首页（介绍 yimsg），聊天 App 挂载在 `/chat/`（浏览器打开域名先看到介绍页，点击进入才是聊天界面） |
| 进程管理 | systemd unit `/etc/systemd/system/yimsg.service`，已 `enable`，开机自启 + 崩溃自动重启 |
| 低端口绑定方式 | `setcap cap_net_bind_service=+ep` 加在二进制上，systemd unit 内 `AmbientCapabilities=CAP_NET_BIND_SERVICE`，不需要以 root 运行 |

## 3. 配置

服务器上 `/opt/yimsg/config.toml`（相对 `config.toml` 模板只覆盖必要项，其余用内置默认值）：

```toml
[server]
host = "0.0.0.0"
port = 443
tls_cert = "/etc/ssl/certs/yimsg.pem"
tls_key = "/etc/ssl/certs/yimsg.key"

[database]
data_dir = "/opt/yimsg/data"
shard_count = 4

[frontend]
static_dir = "/opt/yimsg/web"
mount_path = "/chat/"

[website]
static_dir = "/opt/yimsg/website"
mount_path = "/"

[media]
upload_dir = "/opt/yimsg/data/media"
```

systemd unit `/etc/systemd/system/yimsg.service`：

```ini
[Unit]
Description=Yimsg Server
After=network.target

[Service]
Type=simple
User=yimsg
Group=yimsg
WorkingDirectory=/opt/yimsg
ExecStart=/opt/yimsg/server /opt/yimsg/config.toml
Restart=on-failure
RestartSec=3
AmbientCapabilities=CAP_NET_BIND_SERVICE
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target
```

## 4. 首次部署流程

在本机（macOS/Linux 开发机）执行：

```bash
# 1. 交叉编译 Linux/amd64 二进制（modernc.org/sqlite 是纯 Go 实现，不依赖 CGO，可直接跨平台编译）
GOOS=linux GOARCH=amd64 go build -o server-linux-amd64 ./cmd/server

# 2. 构建前端产物（复用 tools/build.sh 的前半段：协议生成物 + npm run build）
go run ./tools/cmd/protocolgen
(cd frontend && npm run build)   # 产物在仓库根目录 web/

# 3. 传输到服务器（website/ 是仓库自带的静态官网，无需构建）
scp server-linux-amd64 yimsg-server:/opt/yimsg/server
# /opt/yimsg/config.toml 使用第 3 节的生产 HTTPS 配置，不直接覆盖为本地模板
scp -r web/* yimsg-server:/opt/yimsg/web/
scp -r website/* yimsg-server:/opt/yimsg/website/
```

在服务器上执行（首次需要创建账号、目录、systemd unit，SSH 密钥登录已配置好之后免密执行）：

```bash
useradd --system --home-dir /opt/yimsg --shell /usr/sbin/nologin yimsg
mkdir -p /opt/yimsg/web /opt/yimsg/website
# 确认证书已存在并且 yimsg 进程可读；key 不进入版本库。
test -r /etc/ssl/certs/yimsg.pem
test -r /etc/ssl/certs/yimsg.key
chmod +x /opt/yimsg/server
setcap 'cap_net_bind_service=+ep' /opt/yimsg/server
chown -R yimsg:yimsg /opt/yimsg
# 把第 3 节的 systemd unit 内容写入 /etc/systemd/system/yimsg.service
systemctl daemon-reload
systemctl enable --now yimsg
```

## 5. 后续更新部署流程

代码有变更后，重复第 4 节的“本机编译 + 传输”部分（`web/`、`website/` 直接覆盖即可）。**二进制不要直接覆盖同名文件**——传成 `server.new`，在服务器上原子替换，避免覆盖到正在运行的进程文件：

```bash
scp server-linux-amd64 yimsg-server:/opt/yimsg/server.new
ssh yimsg-server "
  chmod +x /opt/yimsg/server.new
  setcap 'cap_net_bind_service=+ep' /opt/yimsg/server.new
  mv /opt/yimsg/server.new /opt/yimsg/server
  chown -R yimsg:yimsg /opt/yimsg
  systemctl restart yimsg
"
```

`setcap` 必须每次对新二进制重新执行——capability 是文件扩展属性，替换文件后会丢失。

## 7. 边界

- **生产使用 HTTPS，端口 443 由 Go 服务直接监听**。云厂商安全组需放行 443（HTTPS）和 22（SSH）；如需保留 HTTP 80 或做 HTTP → HTTPS 跳转，需要后续增加反向代理或独立跳转服务，当前服务进程只监听一个端口。
- **Cloudflare Origin CA 证书只适合放在 Cloudflare 后面使用**。直接用 IP 访问源站做健康检查时，使用 `curl -k` 仅验证源站服务可用；公网用户应通过绑定到 Cloudflare 的域名访问 HTTPS。
- **单机架构，不支持水平扩展**：在线连接注册表、消息 fanout 任务队列都是进程内存态，多实例部署需要额外的跨进程状态同步机制，当前未实现。
- **防火墙/安全组完全由云厂商控制台管理**：服务器本机未安装 `ufw`/`iptables`，端口放行只能去控制台操作，本文档和部署脚本都不涉及。
- **root 密码类凭证不进版本库、不进聊天记录**：SSH 访问统一用密钥（`~/.ssh/yimsg_deploy`，仅本机保存），密码仅用于首次装公钥或紧急救援，用完建议立即在控制台重置。

## 8. 维护点

1. 如未来改为 Caddy / Nginx 终止 TLS：`config.toml` 的 `server.port` 改为内部端口（如 `38081`），清空 `tls_cert` / `tls_key`，由反向代理监听 80/443 并转发到内部 HTTP 端口。
2. 服务器重建或更换 IP 后：更新本文第 2 节的 IP，更新本机 `~/.ssh/config` 里 `yimsg-server` 的 `HostName`。
3. `data/` 目录是唯一状态，迁移/备份服务器时只需要打包这个目录（`config.toml`、`server`、`web/` 都可以从仓库重新生成）。
